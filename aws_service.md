AWS의 서비스 대부분은 엘라스틱 *** 이름으로 되어있음

AWS 콘솔의 대부분 기능은 CLI / API 로 구현가능

## 기본 서비스

주로 쓰이는 서비스들

- EC2 → 클라우드 컴퓨팅 시스템
    - 라우트 53 → DNS 서비스
    - ELB → 로드 벨런서
- RDS → RDBMS 서비스
- S3 → 파일 저장소
    - 퍼블릭 설정과 도메인 연결로 static 파일을 서비스 가능

## 알아두면 좋은 서비스

- EKS → 쿠버네티스 서비스
- ECR → 도커 이미지 저장소

## 함께 쓰면 좋은 서비스

- Cloud Front → CDN 서비스로 컨텐츠 캐싱에 사용
- Cloud watch → 모니터링 서비스로 오토스케일링 등과 함께 사용
- Athena → S3의 데이터들을 쿼리문으로 조회 가능
- Kinesis → 실시간 데이터 스트림 서비스
- Lambda → 소스만 띄우는 서버 리스 서비스
- API Gateway → API와 서비스를 연결해주는 서비스로 Lambda와 함께 많이 사용
- CodeDeploy → 코드 배포를 진행해주는 서비스
- Elastic IP
    - AWS의 서버들은 유동 IP로 서버가 재시작되면 IP가 변경됨
    - 변경되지 않도록 고정 IP를 연결시켜주는 서비스
- ElastiCache → 레디스와 같은 캐싱 시스템을 제공하는 서비스

## 설정시 많이 쓰이는 용어

- 온디멘드 → 사용한 만큼만 과금
- 고가용성 → 장애 발생 없이 오랜시간 사용가능 함
- 인스턴스 유형 → 클라우드 서버의 스펙
    - [인스턴스 유형](https://aws.amazon.com/ko/ec2/instance-types/)
- VPC
    - 서비스의 대역을 나누어 장애 네트워크 장애 발생시 
    영향도가 낮게 설계 할수 있게 하는 설정

## 설정시 주의해야할 사항들

- 모든 서비스는 과금이 된다. 사용하지 않는 서비스는 과감히 삭제하거나 종료시키자.
- 네트워크 비용 또한 과금 대상이나 내부 통신은 과금이 없거나 낮으므로 내부 통신을 이용하자.
- 설정이 반영되는 데에는 서비스가 중단되는 것들도 있다. ex) RDS
- 서비스의 버전이 현저히 낮을 경우 강제 업데이트가 진행된다. ex) EKS,RDS
